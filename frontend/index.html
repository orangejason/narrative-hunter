## 核心逻辑：叙事狙击流水线（Pipeline）
═══════════════════════════════════════════════════════════════

  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
  │ 1. 采集  │ →  │ 2. 爬榜  │ →  │ 3. 反推  │ →  │ 4. 造币  │
  │ 四源并行 │    │ 热点代币 │    │ 叙事角度 │    │ 一键发射 │
  └──────────┘    └──────────┘    └──────────┘    └──────────┘

  Step 1 — 数据采集（四源并行实时拉取）
  ├─ OKX OnchainOS: DEX 行情 + Trending + 持仓分析
  ├─ Binance: Social Rush + Hype + Smart Money
  ├─ 6551 Twitter: KOL 推文 + 社交信号
  └─ Bitget: 涨跌幅排行 + 新币追踪

  Step 2 — 爬热点币榜单
  ├─ 聚合四源数据，提取当前市场真正在涨的代币
  ├─ 按热度/涨幅/交易量/聪明钱流入 多维排序
  └─ 去重 & 交叉验证（同一代币多源确认）

  Step 3 — 反向搜索叙事（核心差异化能力）
  ├─ 对热门代币进行 Twitter KOL 讨论反查
  ├─ 分析 Binance Social Rush 中的关联叙事
  ├─ 提炼叙事角度 & 事件触发因子
  └─ 热度评分算法：
      heat = w1×社交讨论量 + w2×价格变动幅度
           + w3×交易量增速 + w4×聪明钱流入量
      (w1=0.3, w2=0.25, w3=0.25, w4=0.2)

  Step 4 — 基于叙事生成代币方案
  ├─ 从热门叙事中提炼 meme coin 创意
  ├─ 生成：Symbol + Name + 中文描述（带 emoji，有冲击力）
  ├─ 通过 6551 Twitter API 搜索高热度图片作为 Logo
  └─ 一键提交到 Jumpbot.ai 发射器

  ⚠️ 关键原则：不凭空构造热点！
     必须基于实际市场数据 → 反推叙事 → 再造币

## 输出结构定义
═══════════════════════════════════════════════════════════════

  热点叙事卡片 {
    title: string          // 叙事标题（中文）
    description: string    // 详细描述 + emoji
    heat: number           // 热度评分 0-100
    kols: string[]         // 关联 KOL Twitter 账号
    tokens: [{             // 关联代币列表
      symbol, price, change24h, volume, contractAddress
    }]
    source: string         // 叙事来源标识
    timestamp: ISO8601
  }

  热门代币排行 {
    symbol, name, price, change24h, volume24h,
    marketCap, liquidity, holders, chain, logo,
    source: "onchain_os" | "binance" | "bitget" | "ave"
  }

  聪明钱监控 {
    wallets: [{ address, label, pnl, avatar }]
    smartMoneyTokens: [{
      symbol, price, change24h, marketCap, volume,
      smartMoneyInflow, logo
    }]
  }

  发币建议 {
    symbol, name, description, narrativeSource, heat
  }

## 系统约束 & 质量保证
═══════════════════════════════════════════════════════════════

  ✅ 所有叙事描述必须使用中文，带 emoji，有冲击力
  ✅ 不凭空构造热点 — 必须基于实际链上 & 社交数据反推叙事
  ✅ 热度评分基于社交讨论量 + 价格变动 + 交易量 + 聪明钱流入

  ✅ 自动去重 — 已提交代币通过 submitted.txt 防止重复
  ✅ 回测验证 — 每轮生成后通过 DexScreener + Bitget 回测命中率
  ✅ 数据刷新频率：叙事 5min / 代币 1min / 聪明钱 3min
